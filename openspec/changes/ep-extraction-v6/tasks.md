**Post-review CI failure and fix**: after judgment-day approval, GitHub Actions' `yarn build` step failed on `@flash-global66/g-form:build:types` with `TS2305: Module "lodash-unified" has no exported member 'get'/'set'/'clone'/...`. **Process note for future WUs removing a widely-used dependency**: this WU needed 2 rounds of CI-only-reproducible fixes (`lodash-es`/`@types/lodash-es`, then `dayjs`) despite passing 2 independent judgment-day reviews and every local check available at the time. Both failures shared the same shape: a DS package declared something as a peer/transitive expectation that was, in practice, only ever satisfied because `element-plus` also happened to depend on it directly. Lesson recorded for future dependency-removal work: grep every remaining direct dependency's own `peerDependencies` (not just its `dependencies`) for anything the removed package also used to provide, and validate with the EXACT CI build command (`yarn build` via Lerna, not just `vue-tsc`/`vitest`/`--immutable` checks) on a genuinely fresh install before considering a removal complete.
